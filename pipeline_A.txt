PURPOSE: Process demo/sales call transcript → produce v1 Account Memo + v1 Agent Spec

TRIGGER
  ├── Type: Manual Trigger (click to run)
  └── Future: Webhook (receive account_id + transcript path)

     │
     ▼

NODE 1: SET VARIABLES
  ├── account_id: "bens_electric"
  ├── input_path: "/inputs/bens_electric/demo_transcript.txt"
  ├── output_path: "/outputs/accounts/bens_electric/v1/"
  └── version: "v1"

     │
     ▼

NODE 2: READ TRANSCRIPT FILE
  ├── Type: Read Binary File (or Read/Write File node)
  ├── Input: demo_transcript.txt
  ├── Output: Full transcript text as string
  └── Error handling: If file not found → stop workflow, log error

     │
     ▼

NODE 3: READ METADATA FILE
  ├── Type: Read/Write File node
  ├── Input: /inputs/bens_electric/metadata.json
  └── Output: Account metadata (company name, source info)

     │
     ▼

NODE 4: CHUNK CHECK (Decision)
  ├── If transcript < 25,000 tokens → proceed to Node 5 (single call)
  ├── If transcript > 25,000 tokens → go to Node 4a (chunking)
  └── Purpose: Groq has context window limits

     │ (if needs chunking)
     ▼

NODE 4a: CHUNK TRANSCRIPT
  ├── Type: Function/Code node
  ├── Logic: Split transcript into overlapping chunks (~20K tokens each)
  ├── Overlap: 500 tokens between chunks (prevent losing context at boundaries)
  └── Output: Array of chunks

     │
     ▼

NODE 5: LLM EXTRACTION — ACCOUNT MEMO
  ├── Type: HTTP Request node
  ├── URL: https://api.groq.com/openai/v1/chat/completions
  ├── Method: POST
  ├── Headers:
  │   ├── Authorization: Bearer {{ $env.GROQ_API_KEY }}
  │   └── Content-Type: application/json
  ├── Body:
  │   ├── model: "llama-3.3-70b-versatile"
  │   ├── temperature: 0.1 (low = deterministic, less creative)
  │   ├── max_tokens: 4000
  │   ├── response_format: { type: "json_object" }
  │   ├── messages:
  │   │   ├── system: [extraction_demo.md prompt]
  │   │   └── user: "Transcript:\n\n{{ transcript_text }}"
  │   └── Note: If chunked, process each chunk then merge
  └── Output: Raw JSON response from Groq

     │
     ▼

NODE 6: PARSE AND VALIDATE MEMO
  ├── Type: Function/Code node
  ├── Logic:
  │   ├── Parse JSON from LLM response
  │   ├── Validate against account_memo_schema
  │   ├── Check for hallucination indicators:
  │   │   ├── Are there values that weren't in transcript?
  │   │   ├── Are null fields actually null (not invented)?
  │   │   └── Is questions_or_unknowns populated?
  │   ├── Add metadata:
  │   │   ├── generated_at: current timestamp
  │   │   ├── generated_by: "pipeline_a"
  │   │   └── version: "v1"
  │   └── If validation fails → flag for manual review
  └── Output: Validated account_memo.json

     │
     ▼

NODE 7: SAVE ACCOUNT MEMO v1
  ├── Type: Write File node
  ├── Path: /outputs/accounts/bens_electric/v1/account_memo.json
  ├── Content: Validated memo JSON (pretty-printed)
  └── Overwrite: true

     │
     ▼

NODE 8: LLM GENERATION — AGENT SPEC
  ├── Type: HTTP Request node
  ├── URL: https://api.groq.com/openai/v1/chat/completions
  ├── Body:
  │   ├── model: "llama-3.3-70b-versatile"
  │   ├── temperature: 0.2
  │   ├── max_tokens: 6000
  │   ├── response_format: { type: "json_object" }
  │   ├── messages:
  │   │   ├── system: [agent_spec_generator.md prompt]
  │   │   └── user: "Account Memo:\n\n{{ account_memo_json }}"
  │   └── Note: This uses the MEMO as input, not the transcript
  └── Output: Agent spec JSON

     │
     ▼

NODE 9: VALIDATE AGENT SPEC
  ├── Type: Function/Code node
  ├── Logic:
  │   ├── Parse and validate against agent_spec_schema
  │   ├── Verify system_prompt contains required flows
  │   ├── Verify key_variables match memo data
  │   ├── Verify limitations reflect actual unknowns from memo
  │   └── Add metadata
  └── Output: Validated agent_spec.json

     │
     ▼

NODE 10: SAVE AGENT SPEC v1
  ├── Type: Write File node
  ├── Path: /outputs/accounts/bens_electric/v1/agent_spec.json
  └── Content: Validated spec JSON

     │
     ▼
