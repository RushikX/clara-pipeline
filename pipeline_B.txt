PURPOSE: Process onboarding call + existing v1 → produce v2 Memo + v2 Agent Spec + Changelog

  ├── Type: Manual Trigger
  └── Prerequisite: Pipeline A must have run successfully (v1 files must exist)

     │
     ▼

NODE 1: SET VARIABLES
  ├── account_id: "bens_electric"
  ├── onboarding_path: "/inputs/bens_electric/onboarding_transcript.txt"
  ├── v1_memo_path: "/outputs/accounts/bens_electric/v1/account_memo.json"
  ├── output_path: "/outputs/accounts/bens_electric/v2/"
  └── version: "v2"

     │
     ▼

NODE 2: READ ONBOARDING TRANSCRIPT
  ├── Type: Read File node
  ├── Input: onboarding_transcript.txt
  └── Output: Full onboarding transcript as string

     │
     ▼

NODE 3: READ V1 ACCOUNT MEMO
  ├── Type: Read File node
  ├── Input: /outputs/accounts/bens_electric/v1/account_memo.json
  ├── Output: v1 memo as JSON object
  └── Error: If v1 doesn't exist → stop workflow, create error issue

     │
     ▼

NODE 4: LLM EXTRACTION — ONBOARDING UPDATES
  ├── Type: HTTP Request node (Groq API)
  ├── System prompt: extraction_onboarding.md
  ├── User message: BOTH the v1 memo AND the onboarding transcript
  ├── Purpose: Extract ONLY new/changed/confirmed information
  ├── Key instruction in prompt:
  │   ├── "Here is the existing v1 memo from the demo call"
  │   ├── "Here is the onboarding call transcript"
  │   ├── "Return ONLY fields that are NEW, CHANGED, or CONFIRMED"
  │   ├── "For unchanged fields, do not include them"
  │   ├── "For each changed field, include the reason from the transcript"
  │   └── "Update questions_or_unknowns: remove resolved, keep unresolved"
  └── Output: Patch/updates JSON

     │
     ▼

NODE 5: MERGE v1 + PATCHES → v2 MEMO
  ├── Type: Function/Code node
  ├── Logic:
  │   ├── Deep clone v1 memo
  │   ├── Apply each patch from Node 4 output
  │   ├── For each patched field:
  │   │   ├── Update the value
  │   │   ├── Update confidence to "CONFIRMED_BY_CLIENT" where applicable
  │   │   └── Track the change for changelog
  │   ├── Update questions_or_unknowns:
  │   │   ├── Remove items that were resolved in onboarding
  │   │   └── Add any NEW unknowns that emerged
  │   ├── Update metadata:
  │   │   ├── version: "v2"
  │   │   ├── sources: ["demo_sales_call", "kickoff_onboarding_call"]
  │   │   └── generated_at: current timestamp
  │   └── Preserve ALL unchanged fields exactly as they were in v1
  └── Output: Complete v2 account_memo.json

     │
     ▼

NODE 6: GENERATE CHANGELOG
  ├── Type: Function/Code node (NOT LLM — deterministic comparison)
  ├── Logic:
  │   ├── Load v1 memo and v2 memo
  │   ├── Deep compare every field
  │   ├── For each difference:
  │   │   ├── Record field path (dot notation)
  │   │   ├── Record v1 value
  │   │   ├── Record v2 value
  │   │   └── Record reason (from patch data)
  │   ├── Calculate:
  │   │   ├── total_fields_changed
  │   │   ├── total_unknowns_resolved
  │   │   └── total_unknowns_remaining
  │   ├── List unknowns_resolved
  │   └── List unknowns_still_open
  └── Output: changelog.json

     │
     ▼
